# AGI-吉尔伽美什：本地快速开始

这份文档只讲一件事：在本机把核心功能真实跑起来。

当前本地模式使用 Docker 启动 Spark Master、Spark Worker、Spark History Server。真正的 Spark 作业通过容器内 `spark-submit` 执行，不依赖宿主机 Java。

## 1. 本地要求

需要：

- macOS / Linux
- Python 3.9+
- Docker Desktop 或 Docker Engine
- 可以访问 `docker compose`

不需要：

- 宿主机安装 Java
- 宿主机安装 Spark
- 远程 Spark 集群
- 默认启动 Hive

Hive 是可选的，只有你要测试 Hive 表时才启动。

## 2. 安装项目依赖

在项目根目录执行：

```bash
bash scripts/bootstrap.sh
```

激活虚拟环境：

```bash
. .venv/bin/activate
```

安装 Spark 相关 Python 依赖：

```bash
python -m pip install -e ".[spark]"
```

## 3. 生成真实测试数据

```bash
.venv/bin/python scripts/generate-real-demo-data.py
```

会生成这些本地真实数据：

```text
data/users.csv
data/devices.csv
data/merchants.csv
data/transactions.csv
data/user_events.csv
data/graph_edges.csv
data/risk_labels.csv
data/kb/risk_playbook.md
data/training/risk_cases.jsonl
```

当前数据规模大约是：

```text
transactions: 36000
user_events: 18000
graph_edges: 127407
users: 1200
devices: 520
merchants: 160
```

## 4. 启动本地 Spark

```bash
bash scripts/local-stack.sh up
```

默认只启动：

- Spark Master
- Spark Worker
- Spark History Server

访问地址：

```text
Spark Master UI:  http://localhost:8080
Spark Worker UI:  http://localhost:8081
Spark History:    http://localhost:18080
```

确认容器已启动：

```bash
docker ps --format '{{.Names}}' | grep sparkos-spark
```

## 5. 启动 TUI

```bash
bash scripts/run-tui.sh config/local.yaml
```

`config/local.yaml` 是本地真实数据演示配置，包含：

- 本地真实数据 Catalog
- Docker Spark 容器提交配置
- Spark History Server 地址
- 本地 artifact 输出路径
- 向量知识库本地存储路径

## 6. 跑通真实 Spark SQL

在 TUI 输入：

```text
统计 transactions 表最近 7 天每个用户的交易次数并执行
```

预期结果：

- Agent 接管任务
- 生成分布式查询计划
- 通过 Docker 容器执行真实 `spark-submit`
- 生成 DAG 诊断
- 产物写入 `artifacts/agent-runs/<run-id>/`

重点检查这个文件：

```bash
cat artifacts/agent-runs/<run-id>/06_execution_config.json
```

里面应该能看到：

```json
{
  "status": "success",
  "metrics": {
    "execution_mode": "docker_spark",
    "return_code": 0,
    "spark_app_id": "app-...",
    "input_rows": 36000,
    "preview_rows": 20
  }
}
```

真实 Spark 结果文件：

```bash
cat artifacts/agent-runs/<run-id>/docker_spark_result.json
```

## 7. 跑通 ETL / 数据质量 / DAG 诊断

在 TUI 输入：

```text
构建用户日活宽表 ETL 并检查数据质量
```

预期产物：

```text
artifacts/agent-runs/<run-id>/02_etl_pipeline.json
artifacts/agent-runs/<run-id>/04_data_quality.json
artifacts/agent-runs/<run-id>/06_execution_config.json
artifacts/agent-runs/<run-id>/07_dag_diagnosis.json
```

展示重点：

- 自然语言拆成多阶段任务
- 自动选择 ETL、质量检查、执行、诊断 skill
- Spark 执行配置和诊断产物可审计

## 8. 跑通图计算

在 TUI 输入：

```text
分析 graph_edges 用户设备关系网络，找出可疑团伙社区
```

预期产物：

```text
artifacts/agent-runs/<run-id>/08_graph_result.json
```

这个任务会读取真实的：

```text
data/graph_edges.csv
```

结果里应该能看到类似：

```json
{
  "metrics": {
    "execution_mode": "in_process_graph",
    "edge_count": 5000,
    "vertex_count": 4948
  }
}
```

说明：当前图算法在本进程执行，输入来自真实生成的关系边数据。

## 9. 跑通向量知识库

在 TUI 输入：

```text
构建向量知识库 @data/kb/risk_playbook.md
```

预期产物：

```text
artifacts/vector-kb/chunks.jsonl
artifacts/vector-kb/manifest.json
```

## 10. 跑通训练数据处理

在 TUI 输入：

```text

```

预期产物：

```text
artifacts/training-data/training_dataset.jsonl
artifacts/training-data/manifest.json
```

- `@文件` 触发任务处理模式

## 11. 普通对话

在 TUI 输入：

```text
你好，介绍一下你自己
```

预期：

- 不进入任务处理模式
- 正常流式对话
- 顶部状态栏显示模型名和 used token

## 12. 可选启动 Hive

默认不启动 Hive，因为镜像较大。

如果确实要测试 Hive demo 表：

```bash
bash scripts/local-stack.sh hive-up
```

会初始化：

```text
sparkos_demo.transactions
sparkos_demo.users
```

```bash

```

```bash
bash scripts/local-stack.sh status
```


## 15. 最小演示顺序

建议现场按这个顺序演示：

1. 启动 Docker Spark：`bash scripts/local-stack.sh up`
2. 启动 TUI：`bash scripts/run-tui.sh config/local.yaml`
3. 普通对话：`你好，介绍一下你自己`
4. 真实 Spark：`统计 transactions 表最近 7 天每个用户的交易次数并执行`
5. 图计算：`分析 graph_edges 用户设备关系网络，找出可疑团伙社区`
6. 向量知识库：`构建向量知识库 @data/kb/risk_playbook.md`
7. 训练数据：`请做训练数据处理 @data/training/risk_cases.jsonl`

一句话总结：

用户只和 TUI 里的 AGI-吉尔伽美什对话；系统自动识别普通对话、文件任务、大数据任务和图计算任务。真实 Spark 作业通过 Docker 容器内 `spark-submit` 执行，结果和诊断统一写入 `artifacts/`。
